Class org.osgi.service.provisioning.ProvisioningService
Service for managing the initial provisioning information.
addInformation(Dictionary) Tests:
Adds the key/value pairs contained in info to the Provisioning Information dictionary. testDictionary Check the dictionary manipulations. Mainly the fact that the dictionary should not be mutable.
testIPAReference Test if an ipa file that contains a PROVISIONING_REFERENCE is correctly loading the first + referred file.
testFileLoad Test if it is possible to load a provisioning jar setting the PROVISIONING_REFERENCE to a file: URL. This file, simple.ipa contains a bundle and it has the start reference set. The simple.jar bundle should start.
testFileWithReference Test a file that only references a JAR by a URL
testCaseReference Load an ipa file with a reference that uses an unusal case for the mime type. According to rfc 2046, mime types are not case sensitive. There is a mistake in the spec because it refers to the mime type with upper and lower case URL/url. See issue #175
addInformation(ZipInputStream) Tests:
Processes the ZipInputStream and extracts information to add to the Provisioning Information dictionary, as well as, install/update and start bundles. testDictionary Check the dictionary manipulations. Mainly the fact that the dictionary should not be mutable.
getInformation() Tests:
Returns a reference to the Provisioning Dictionary. testDictionary Check the dictionary manipulations. Mainly the fact that the dictionary should not be mutable.
testPersistence Check if the changes to the dictionary are persistent. We will test this by stopping the provisioning service and verifiying the update count and a key set to the time.
testStartAndRef Figure 78 allows both the key provisioning.start.bundle and provisioning.reference to be set in the same same go. So ipa-ref-start refers to another ipa file (simple.ipa) that loads a bundle, but also loads and starts a local bundle.
testSPID Test if the provisioning service sends the SPID to the server. we have registered a "spid-test" url handler that we use. It will store the query part of a URL in the query field.
testSPIDWithQuery Test if our own parameters are maintained when we give a URL where the Provisioning Service will add a SPID.
